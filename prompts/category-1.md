# 类别 1：编码与调试（提示词 1-50）

这些提示词可以让 Claude 变成一个仿佛永不休息、也永不沮丧的高级开发工程师。

1. **代码审查**  
   以高级工程师的视角审查这段代码。识别其中的缺陷、安全漏洞、性能问题以及代码风格问题。解释每个问题，并给出修复建议：`[粘贴代码]`

2. **调试这个错误**  
   我遇到了这个错误：`[错误信息]`。下面是相关代码：`[粘贴代码]`。请带我逐步分析所有可能原因，并说明如何逐一修复。

3. **重构以提升可读性**  
   在不改变功能的前提下，重构这段代码，使其更加整洁、易读。为复杂逻辑添加注释说明：`[粘贴代码]`

4. **编写单元测试**  
   为这个函数编写完整的单元测试。覆盖正常路径、边界情况、空输入以及错误场景：`[粘贴函数]`

5. **优化性能**  
   分析这段代码中的性能瓶颈。找出最慢的部分，并将其改写得更高效：`[粘贴代码]`

6. **转换为 TypeScript**  
   将这段 JavaScript 代码转换为 TypeScript。为所有变量、参数和返回值添加合适的类型定义：`[粘贴代码]`

7. **解释这段代码**  
   用面向初学者的方式向我解释这段代码。它是做什么的、如何工作的，以及为什么要这样写：`[粘贴代码]`

8. **生成 API 接口**  
   使用 `[语言/框架]` 编写一个 REST API 接口，用于实现 `[描述它应该完成的功能]`。需要包含输入校验、错误处理以及正确的 HTTP 状态码。

9. **数据库查询优化**  
   这条 SQL 查询执行得很慢：`[粘贴查询语句]`。请分析其缓慢原因，解释问题所在，并将其重写得更快。同时建议哪些索引会有帮助。

10. **编写文档**  
    为这个函数编写清晰的开发者文档，包括用途、参数、返回值、使用示例以及边界情况：`[粘贴函数]`

11. **安全审计**  
    审计这段代码中的安全漏洞。检查 SQL 注入、XSS、认证问题、泄露的密钥以及其他潜在风险：`[粘贴代码]`

12. **构建 CLI 工具**  
    使用 Python 编写一个命令行工具，实现 `[描述功能]`。需要包含参数解析、错误处理、帮助说明以及使用示例。

13. **将函数改为异步**  
    将这个同步函数改写为 async/await 形式。正确处理所有 Promise 拒绝情况，并保持原有功能不变：`[粘贴函数]`

14. **设计数据模型**  
    为 `[描述应用]` 设计数据库 Schema。包括所有表、字段、数据类型、表关系、索引，并解释你的设计决策。

15. **编写正则表达式**  
    编写一个用于匹配 `[描述要匹配的内容]` 的正则表达式。解释模式的每一部分，并给出匹配与不匹配的示例。

16. **构建网页爬虫**  
    使用 Python 编写一个网页爬虫，从 `[URL/网站类型]` 中提取 `[描述数据]`。需要优雅处理分页、限流和请求失败等情况。

17. **代码架构审查**  
    审查这个架构，并说明在系统规模扩大时你预见到哪些问题。最先会出问题的部分是什么？应该如何重新设计：`[描述或粘贴架构]`

18. **添加错误处理**  
    为这段代码补充完整的错误处理。捕获所有可能失败的情况，进行恰当记录，并优雅降级：`[粘贴代码]`

19. **构建定时任务**  
    使用 `[语言]` 编写一个定时任务，用于执行 `[描述任务]`。它应当按 `[执行频率]` 运行，能够处理失败，并在异常发生时发送告警。

20. **跨语言翻译代码**  
    将这段代码从 `[语言 A]` 翻译成 `[语言 B]`。保留全部功能，并使用目标语言中符合惯例的写法：`[粘贴代码]`

21. **编写 Webhook 处理器**  
    使用 `[框架]` 编写一个 Webhook 处理器，用于接收来自 `[服务]` 的事件。它需要校验签名、解析载荷，并处理 `[描述事件]`。

22. **构建限流器**  
    使用 `[语言]` 实现一个限流器，要求每个用户在 `[时间周期]` 内最多允许 `[X]` 次请求。需要处理边界情况，并支持将某些用户加入白名单。

23. **创建中间件**  
    为 `[框架]` 编写一个中间件，实现 `[描述它应该做什么]`。要求可复用、文档清晰，并处理所有边界情况。

24. **设计缓存策略**  
    为 `[描述系统]` 设计一个缓存策略。哪些内容应该缓存？缓存多久？何时失效？应使用什么缓存技术？

25. **编写数据迁移脚本**  
    编写一个数据库迁移脚本，实现 `[描述需要完成的任务]`。要求幂等、可回滚，并且能够安全地在生产数据库上运行。

26. **构建认证系统**  
    使用 `[框架]` 设计并编写一个基于 JWT 的认证系统。包括注册、登录、令牌刷新、登出以及受保护路由的中间件。

27. **生成模拟数据**  
    编写一个脚本，为 `[描述数据结构]` 生成逼真的模拟数据。生成 `[数量]` 条记录，并以 JSON 格式输出。

28. **实现分页**  
    为这个 API 接口添加基于游标的分页。包括分页大小限制、边界情况处理以及正确的元数据返回：`[粘贴接口代码]`

29. **编写 CI/CD 流水线**  
    编写一个 GitHub Actions 工作流，在每次推送到 main 分支时执行测试、检查代码质量、构建应用并部署到 `[平台]`。

30. **设计微服务**  
    设计一个用于处理 `[描述功能]` 的微服务架构。它需要哪些接口？如何与其他服务通信？其数据模型应如何设计？

31. **实现重试逻辑**  
    为这个调用外部 API 的函数添加重试逻辑。需要处理限流、瞬时错误，并使用指数退避：`[粘贴函数]`

32. **编写状态机**  
    为 `[描述具有状态的系统]` 实现一个状态机。定义所有状态、状态转换、守卫条件和执行动作。

33. **构建队列系统**  
    使用 `[语言]` 实现一个任务队列，用于处理 `[描述任务]`。需要包含优先级、重试逻辑、死信处理以及队列深度监控方式。

34. **创建 SDK**  
    为以下 API 设计并编写一个基础 SDK，使用 `[语言]` 实现。要求易于使用、文档完善，并自动处理认证：`[粘贴 API 规范]`

35. **优化 React 组件**  
    这个 React 组件渲染过于频繁，导致性能问题。请分析其原因并修复性能问题：`[粘贴组件代码]`

36. **编写集成测试**  
    为这个 API 接口编写集成测试，覆盖包含数据库在内的完整请求/响应流程：`[粘贴接口代码]`

37. **实现搜索功能**  
    使用 `[技术]` 为 `[描述数据]` 实现全文搜索。包括模糊匹配、相关性评分，以及按 `[字段]` 进行过滤。

38. **构建通知系统**  
    设计并实现一个通知系统，在 `[描述触发条件]` 时发送 `[邮件/SMS/推送]` 通知。需要包括模板、队列和投递跟踪。

39. **编写解析器**  
    使用 `[语言]` 编写一个解析器，用于读取 `[描述格式]` 并输出 `[描述输出结果]`。需要优雅处理格式错误的输入。

40. **设计事件系统**  
    为 `[描述应用]` 设计一个事件驱动架构。应该有哪些事件？事件结构应如何设计？服务之间应如何通过事件通信？

41. **实现文件上传**  
    使用 `[框架]` 编写一个文件上传处理器，接收 `[文件类型]`，校验大小和类型，将文件存储到 `[存储位置]`，并返回访问 URL。

42. **构建配置系统**  
    为 `[应用类型]` 设计一个配置管理系统，支持多环境、密钥管理，以及无需重启即可在运行时更新配置。

43. **编写健康检查接口**  
    为 `[框架]` 编写一个健康检查接口，用于检查数据库连通性、外部服务可用性以及系统资源状态。返回合适的 HTTP 状态码。

44. **实现功能开关**  
    设计并实现一个功能开关系统，支持按用户、按流量百分比或全局启用/禁用某项功能，并且无需重新部署代码。

45. **创建日志系统**  
    为 `[应用类型]` 实现结构化日志系统。应包含请求 ID、用户上下文、性能耗时，并集成到 `[日志服务]`。

46. **构建报表生成器**  
    编写代码，从 `[描述数据源]` 查询数据，并生成一个 `[格式]` 报表，展示 `[描述指标]`。安排其按 `[执行频率]` 运行，并通过邮件发送给 `[收件人]`。

47. **实现数据校验层**  
    为这个 API 编写一个完整的数据校验层。校验所有输入，返回清晰的错误信息，并防止任何格式错误的数据进入数据库：`[粘贴 API 规范]`

48. **设计插件系统**  
    为 `[应用]` 设计一个插件架构，使第三方开发者无需修改核心代码即可扩展功能。定义插件接口和生命周期钩子。

49. **编写压测脚本**  
    使用 `[工具]` 编写一个压测脚本，模拟 `[数量]` 个并发用户执行 `[描述用户行为]`，并输出响应时间和错误率报告。

50. **构建仪表盘后端**  
    设计并编写一个实时仪表盘后端，用于展示 `[描述指标]`。需要包含 WebSocket 支持、数据聚合以及高效查询能力。
